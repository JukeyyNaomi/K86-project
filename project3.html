Voorwoord

Tijdens het eerste blok heb ik mij voorgenomen om mij te gaan verdiepen in de back-end taal PHP. Met PHP kun je via de server side ervoor zorgen dat bepaalde informatie en interactie die via de gebruiker worden uitgevoerd op het systeem, wordt opgeslagen en daar een passende reactie op plaats vindt. Zo kan je een CMS (Content Management System) maken, wordpress thema’s bouwen en contactformulieren ontwikkelen voor jouw website.

Ik heb gekozen om PHP te doen, omdat ik de laatste tijd veel werk met wordpress. Wordpress is veelal opgebouwd uit PHP. Als je een website host, moet je soms ook met het programma PHP my admin kunnen werken. Ook wou ik altijd al een werkend contactformulier kunnen coderen.

PHP leren

De eerste stap die ik heb ondernomen is het volgen van de cursus PHP op Codecademy. Ik wilde eerst weten hoe makkelijk ik de code oppak, voordat ik echt kon bepalen wat ik ging maken voor mijn deliverable. Tijdens alle lessen kreeg je drie vensters te zien: een uitleg venster, een code editor venster en een uitwerking in de browser. Wanneer je de theoretische hoofdstukken had afgerond, kon je eerst een kleine quiz doen, daarna werd een praktische opdracht voorgelegd die je stapsgewijs kon maken, om zo goed mogelijk de taal te leren.

Oefenen

Verschillende codes uit de oefeningen heb ik bestudeerd en opgeslagen, zodat ik hier altijd naar terug kan kijken tijdens de development fase van mijn project. Heel veel codes heb ik eigenlijk al geoefend door Codecademy, dus was het niet heel moeilijk om te bepalen wat ik wou doen voor mijn project.

Coderen

Uiteindelijk heb ik gekozen om een contactformulier te maken met een soort custom CMS. De gebruiker vult het formulier in, het formulier controleert of de gegevens kloppen en vervolgens worden de gegevens doorgestuurd om een nieuwe post te maken op de pagina. Ik heb gekozen om het concept van de pagina een soort open blog te maken voor mensen die houden van het delen van recepten. De gebruiker voert een recept in en deze komt vervolgens op de website terecht.

Itereren

Ik heb meerdere malen dezelfde code gemaakt, maar elke keer als ik de code opnieuw maakte zorgde ik ervoor dat de code ‘global’ zou worden. Dit houdt in dat je dan niet meer gebonden zou zijn aan een bepaalde ingevoerde variabele en aan de huidige pagina waar je op zit. Je wilt bijvoorbeeld dat de code die is ingevoerd ook zichtbaar is op een productdetail pagina van het recept die de gebruiker zojuist heeft aangemaakt.

Tips

  • Werken en oefenen met Codecademy versnelt en maakt je leerproces efficienter.
  • Ga vele malen itereren. Je code zal misschien niet in 1x perfect zijn, dus neem je tijd en zoek uit hoe je jouw code kan optimaliseren.
  • Kies een onderwerp wat je oprecht interesseert en waar je iets mee wilt doen in de toekomst, dan ben je ook meer gemotiveerd tijdens het project.

Ervaring

Omdat ik geen MySQL database had, konden de gegevens niet opgeslagen worden, dus je kan het formulier invullen, maar de gegevens vervangen zichzelf. De code opzicht werkt hoe het zou moeten werken en daar ben ik erg blij mee.
