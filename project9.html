Voorwoord

Tijdens dit project ben ik een hi-fi prototype gaan coderen doormiddel van html en css. Dit wilde ik graag leren omdat ik in het verleden wel websites heb gecodeerd, en prototypes, maar ik heb nooit zelf de stap gezet van prototype naar coderen. De relevantie was voor mijzelf, omdat ik het bestaande hi-fi prototype van mijn portfolio website heb gekozen om te coderen. Deze kan ik dan blijven updaten en voor toekomstige sollicitaties/opdrachten gebruiken.

Onderzoek

Om te beginnen heb ik mijn kennis van html en css opgefrist door een aantal tutorials, en heb ik de GitHub opgezet en deze gelinked aan VSCode. Hiermee kon ik de basis van de website opzetten, als een soort mal, waar ik later in het project de content van mijn portfolio in kon verwerken.

Schetsen

Aangezien coderen voor mij niks nieuws is, heb ik ervoor gekozen om de content van mijn portfolio op een nieuwe en interessantere manier te displayen dan hoe het er origineel in het prototype staat. Hiervoor heb ik een aantal schetsen gemaakt en ideeÃ«n gebrainstormd, vervolgens kon ik deze gaan doorvoeren en op zoek naar opties om dit uit te werken.

Coderen

Het laatste deel van dit project heb ik besteed aan het coderen, met veel trial and error, van mijn content en de manier waarop ik dit wou displayen. Toen ik alle puntjes op de i had gezet heb ik nog even gecontroleerd of alles in de GitHub ook compleet was, zodat ik deze link kon delen.

Tips

  • Maak een goede planning en gun jezelf wat buffers als dingen langer duren dan verwacht.
  • Hou je code netjes en maak aantekeningen, zo blijft alles overzichtelijk.
  • Werk even ergens anders aan als je vastloopt, zo hou je je hoofd rustig.

Ervaring

Dit project was erg leuk om te doen, vooral omdat ik nu beide kanten van het vakgebied kon samenvoegen. Ik vergeet soms wel hoe lastig een website coderen ook alweer is, en dat heb ik deze weken zeker gemerkt. Het heeft even wat bloed zweet en tranen gekost, maar uiteindelijk ben ik erg tevreden met het resultaat.
